# 链接有效性验证机制说明

## 问题背景

在 2026-04-25 的新闻更新中，发现了一条链接失效的问题：
- **新闻**：英国《数字市场、竞争和消费者法案》(DMCC) 正式通过
- **原链接**：`https://www.gov.uk/government/news/...` → 404 错误
- **原日期**：2026-04-25 → 错误（实际通过时间为 2024-05-24）

## 问题根源分析

### 1. 缺乏链接验证机制
- 添加新闻时未验证链接有效性
- 直接使用搜索结果中的链接，未确认是否可访问
- 没有自动化验证工具

### 2. 日期准确性问题
- 假设新闻是"今天"发布的
- 未仔细核实原始报道的发布日期
- 对于历史事件（如法案通过），未确认实际时间线

## 解决方案

### 1. 新增验证工具

#### 方式一：web_fetch 工具（推荐）
```javascript
// 在添加每条新闻前验证
web_fetch({
  url: "https://example.com/news"
})
// 确保返回 200 状态码且有实际内容
```

**优点**：
- 即时验证，发现问题立即修复
- 可以同时验证内容是否匹配
- 无需额外工具

#### 方式二：Bash 验证脚本
```bash
bash scripts/validate-links.sh
```

**优点**：
- 快速批量验证
- 使用 curl 检查 HTTP 状态码
- 自动生成无效链接报告

**输出示例**：
```
🔍 开始验证新闻链接...

📋 共找到 17 个链接需要验证

[1/17] 检查: https://example.com/news1 ... ✅ OK (200)
[2/17] 检查: https://example.com/news2 ... ❌ FAILED (404)

========================================
✅ 有效链接: 16
❌ 无效链接: 1

需要修复的链接:
   └─ 无效链接: https://example.com/news2
```

#### 方式三：Node.js 验证脚本
```bash
node scripts/validate-links.js
```

**优点**：
- 详细的验证报告
- JSON 格式输出，便于集成
- 包含重定向跟踪

### 2. 更新工作流程

#### 新增强制验证步骤
在 `WORKFLOW.md` 中明确规定：

**内容生成阶段**：
- 每条新增新闻的链接必须先通过验证
- 验证失败的处理流程：
  1. 链接返回 404/403 → 搜索替代链接或删除
  2. 链接超时/无响应 → 重试2次，仍失败则搜索替代
  3. 内容不匹配 → 核实真实性，必要时删除

**构建推送阶段**：
- 构建前必须运行链接验证
- 验证通过才可继续构建
- 验证失败需修复后重新验证

#### 新增日期准确性要求
- 新闻日期必须与原始报道发布日期一致
- 不得使用当前日期填充历史新闻
- 对于历史事件，核实实际时间线

## 使用指南

### 添加新闻时的验证流程

1. **搜索新闻**
   ```bash
   node scripts/search.js "关键词" -n 10
   ```

2. **验证链接（必须）**
   ```javascript
   web_fetch({ url: "候选链接" })
   ```

3. **确认日期**
   - 仔细阅读原文发布日期
   - 不要假设是"今天"发布
   - 对于历史事件，核实实际日期

4. **添加新闻**
   - 确保链接有效
   - 确保日期准确
   - 确保 ID 唯一

5. **批量验证（可选）**
   ```bash
   bash scripts/validate-links.sh
   ```

6. **构建推送**
   ```bash
   npm run build
   git add . && git commit -m "feat: daily update"
   git push origin main
   ```

## 维护建议

1. **定期全量验证**
   - 建议每周运行一次全量验证
   - 及时发现并修复失效链接

2. **监控链接变化**
   - 某些网站可能改版导致链接失效
   - 保持对重要新闻源的监控

3. **建立备份链接库**
   - 对于重要新闻，保存多个来源
   - 主链接失效时可快速替换
